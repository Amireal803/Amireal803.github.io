> 理解系统如何将数据在存储器层次结构中上下移动，了解如何更好更快的访问内存能对自己的程序应用有深度认知以及优化剖析。本文将梳理存储器的层次结构以及虚拟内存的排布结构。

# 1. About Memory Hierarchy

> 存储器系统(Memory System)是一个具有不同容量、成本和访问时间的存储设备的层次结构。

**一个典型的存储器层次结构如下：**
    
- **(L0)寄存器(Registers)：** 直接位于CPU芯片内部，容量极小但速度最快(通常在一个CPU时钟周期内)，用于存储当前参与运算的活跃数据与地址。

- **缓存(Cache)：** 即数据交换的缓冲区，位于CPU和内存之间的小容量高速临时存储器，能够在几个CPU时钟周期内访问，目的是通过存储最可能被CPU再次访问的数据以减少对主存的访问延迟。

    - **(L1)L1高速缓存(SRAM)：** 一级缓存中又分为数据缓存(D-Cache)和指令缓存(I-Cache)，距离CPU核心最近且速度最快，容量最小。
    - **(L2)L2高速缓存(SRAM)：** 位于L1和L3之间且速度稍慢于L1，容量适中。
    - **(L3)L3高速缓存(SRAM)：** 在所有缓存级别中距离核心最远、速度最慢、容量最大。

- **(L4)主存(Main Memory / DRAM)：** 容量远大于缓存但访问速度很慢，当数据不在任何缓存中时CPU必须从主存中获取。

![Memory Hierarchy](/images/memory/memory_hierarchy.png)

**通过这种多级内存层次结构访问内存，我们需要尽可能满足局部性原理来提高效率：**

- **时间局部性 (Temporal Locality)：** 如果一个内存的地址被访问了一次，那么在将来有可能会被多次引用。

- **空间局部性 (Spatial Locality)：** 如果一个内存的地址被访问了一次，那么在将来有可能这个内存附近的内存地址也会被引用。



# 2. Caching in Memory Hierarchy

> 存储器层次结构的中心思想是每一层的缓存来自较低一层的数据对象

**以下图为例：**

- 第 *k+1* 层的存储器被划分成连续的数据对象**组块(chunk)**，称为**块(block)**。每个块都有唯一的地址或名字使之区别于其他块。块**通常是固定大小的**。类似的，第 *k* 层的存储器被划分为更少的块的集合，每个块的大小与 *k+1* 层的块大小相同。任何时刻，第 *k* 层的缓存包含第 *k+1* 层的子集的副本。

- 数据总是以块为大小的**传送单元(transfer unit)** 在第 *k* 层与第 *k+1* 层**来回复制**的

![Fundamental Cache Principle](/images/memory/cache_principle.png)

**一般而言，层次结构中较低层的设备的访问时间越长，因此为了补偿访问过程中浪费的时间，倾向于使用较大的块。**

- **缓存命中(Cache Hit)：** 当程序需要读取第 *k + 1* 层的某个数据对象 *d* 时，*d* 刚好缓存在第 *k* 层的一个块中，这种情况就称为缓存命中。

- **缓存不命中(Cache Miss)：** 相反，若 *d* 没有缓存在第 *k* 层中，这种情况就称为缓存不命中。当发生缓存不命中时，第 *k* 层的缓存从第 *k + 1* 层缓存中取出包含 *d* 的块并以 替换策略(如最不常使用(LFU)、最近最少使用(LRU)、随即替换) 覆盖现存的块。常见的不命中种类有：
    - 强制性未命中 (Compulsory Miss / Cold Miss)：程序首次访问数据时不可避免的Miss。
    - 容量性未命中 (Capacity Miss)：缓存太小，无法容纳所有数据。
    - 冲突性未命中 (Conflict Miss)：不同数据项映射到缓存中的同一位置。



# 3. Inner Structure of DRAM
- DRAM芯片中的单元被分成 *d* 个**超单元(supercell)**，每个超单元能够存储 *ω* bit的数据。即一个 *d × ω* 的DRAM总共存储了*d · ω*位信息 

**以下图被组织成4×4×8阵列的超单元DRAM芯片为例：**

![DRAM chip](/images/memory/DRAM_chip.png)

- 其中每个方格代表一个超单元，每个超单元可以通过类似坐标的方式( *i*, *j* )寻址。信息通过 **地址引脚(addr pin)** 和 **数据引脚(data pin)** 与内存控制器相连

**取出图中蓝色部分超单元( 2, 1 )为例：**

- 首先内存控制器发送 **行地址选通(RAS) 2**到DRAM芯片，DRAM做出响应：将第二行整行数据全部复制到内部的行缓冲区

![DRAM chip RAS](/images/memory/DRAM_chip_RAS.png)

- 接着内存控制器发送 **列地址选通(CAS) 1**到DRAM芯片，DRAM做出响应：将行缓冲区中对应的数据位复制并发送给内存控制器

![DRAM chip CAS](/images/memory/DRAM_chip_CAS.png)

**因此整个过程利用2个地址引脚和8个数据引脚完成数据的访问。**

## 分两次发送地址的原因：

**首先我们要知道芯片的物理体积很大程度上取决于引脚的数量，引脚越少，芯片越小、越便宜、越省电。**
**1个物理引脚在某一瞬间只能传输 1 个二进制位（即 0 或 1）**

- 若将上例中128bit(16×8)DRAM的存储单元用线性排布，那么地址范围( 0 ~ 15 )的二进制位数需要4位( 0000 ~ 1111 )，即需要4个地址引脚。

- 若设计成二维块寻址，在4×4的阵列中，行地址和列地址范围( 0 ~ 3 )的二进制位数各需要2位( 00 ~ 11)，此时只需要2个地址引脚配合 *RAS* 和 *CAS* 控制开关，传输行地址时，RAS控制两个引脚传输行地址的二进制位完成行缓冲区的数据复制，接着复用两个引脚并由CAS控制这两个引脚传输列地址的二进制位。**但不难看出这种分步控制会导致二维阵列的组织方式增加数据访问时间。**

因此存储单元被设计成二维的阵列而不是线性的数组，是以牺牲部分初期访问时间为代价，换取芯片地址引脚数量的大幅降低。

# 4. Memory Modules

- DRAM芯片封装在 **内存模块(Memory Module)** 中

**以下图内存模块的基本组成为例：**

![Memory Module](/images/memory/memory_module.png)

- 该模块一共使用8个DRAM芯片，编号分别用0 ~ 7表示，每个DRAM芯片大小为8M × 8bit = 64Mbit，即8MB，因此整个模块大小为64MB。假设每个超单元可以存储8bit数据，那么对于8字节(64bit)的数据就需要8个超单元来存储，这8个超单元平均分布在8个DRAM芯片上，其中DRAM 0存储低8位，DRAM 1存储下一字节，以此类推。

- 当处理器向内存控制器发起读取8字节数据的请求时，内存控制器将内存地址转换成一个超单元地址 *( i, j )* 发送到内存模块，内存模块将 行地址 *i* 和 列地址 *j* 广播到每个DRAM，每个DRAM输出对应超单元的数据，最后内存模块将所有超单元的数据整合成一个64bit的数据返回给内存控制器。
